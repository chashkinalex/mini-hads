# Roadmap

## Phase 1: Production-ready core

1. Switch local and server environments to PostgreSQL
2. Add public deployment for API and frontend
3. Configure environment separation: local, staging, production
4. Add monitoring, logging, and error tracking

## Phase 2: Real platform auth

1. MAX launch validation
2. Telegram Mini App launch validation
3. VK Mini App launch validation
4. Replace current dev launch flow with production adapters

## Phase 3: Clinical workflow polish

1. Session history for doctors
2. Search and filtering for completed questionnaires
3. Better patient join landing and real deep links
4. Export or printable result summary

## Phase 4: Compliance and operations

1. Privacy policy and legal pages
2. Access control for staff accounts
3. Database backup policy
4. Audit trail for session lifecycle events
